# 微信系列三季重构 · 迁移与规划更新计划

## 目标

把系列从「认知 / skill剖析 / 进阶」重构为「设计思想 / 项目实操 / 细节设计」三季，完成确定性的结构迁移与规划文档更新。本计划只做**结构迁移 + 规划骨架更新**，不写实操季的新文章正文（各篇另走 brainstorm）。

成功标准：
1. 现 `第二季-skill深度剖析/` 的 skill 剖析平移为第三季细节季，目录与文件名统一 `s3-` 前缀。
2. S2-00 全链路总览上移到新建的第二季实操季。
3. 006 顶层总规划的三季叙事翻转为「思想/实操/细节」，对账表与下一步同步。
4. 007 重新定位：进阶/治理内容下沉为细节季专题，不再是"第三季独立路线图"。
5. 新建实操季路线图（场景矩阵）。
6. 全仓交叉引用、README、台账、发布 skill 无断链、无残留旧季号。

## 关键决策（来自 brainstorm + 本轮确认）

| 决策 | 选择 |
|---|---|
| 季结构 | 三季：设计思想（现一季）/ 项目实操（新）/ 细节设计（现 skill 剖析） |
| 文件迁移粒度 | 目录改名 `第三季-skill深度剖析/` + 文件 `s2-` → `s3-` 全改 |
| S2-00 归属 | 上移实操季（不进细节季） |
| 12 篇正文 | 不重写，仅改季号前缀与 frontmatter description 里的"第二季"字样 |
| 图片引用 | `](../pic/` 相对路径，目录平级改名后不变，零改动 |
| BIP | 贯穿番外，不绑定季 |

## 前置事实（已核查）

- 现 `第二季-skill深度剖析/` 共 23 文件：s2-00~s2-11 正文 + outline（s2-11 仅 outline）。
- 图片引用全部为 `](../pic/`，目录改名后相对层级不变 → **无需改图片引用**。
- 外部引用该目录路径的文件：`文章系列/README.md`、`文章系列/已发布台账.md`、`文章系列/运营规划/003-系统总框架.md`。
- 正文/outline 中 24 处"第二季"字样（多在 frontmatter description 与导语），语义指现 skill 季 → 改为"第三季"；S2-00 因上移实操季单独处理。
- 发布 skill `SKILL.md` 选题扫描已是递归 `**/*.md`，目录改名不影响扫描，但 argument-hint 示例与路线图指针需检查。
- 已知历史瑕疵：部分 outline 的 description"第 N 篇"与文件 s2-NN 编号不符（如 s2-02 写"第3篇"、s2-05-outline 写"第10篇"）。**本计划顺手修正为与文件编号一致。**

## 实施步骤

### 步骤 1：目录改名 + 文件前缀迁移（细节季）

> 验证：迁移后 `第三季-skill深度剖析/` 含 21 文件（s3-01~s3-11 正文+outline，扣除上移的 s2-00 两文件）；无 s2- 残留。

1.1 `git mv 文章系列/第二季-skill深度剖析 文章系列/第三季-skill深度剖析`（若文件未入版本控制则用普通 `mv`）。
1.2 把 s2-00 的两个文件（`s2-00-full-journey.md`、`s2-00-full-journey-outline.md`）移出到实操季目录（见步骤 2）。
1.3 其余 s2-01~s2-11（21 文件）就地改名 `s2-` → `s3-`：`s3-01-runtime-setup.md` … `s3-11-optimize-outline.md`。
1.4 验证：`ls 文章系列/第三季-skill深度剖析/ | grep -c '^s2-'` 应为 0。

### 步骤 2：新建实操季目录 + 接收 S2-00

> 验证：实操季目录存在，含上移的总览篇 + 使用说明种子（或其指针）。

2.1 新建目录 `文章系列/第二季-实操案例/`（命名最终以步骤 5 路线图为准）。
2.2 把 s2-00 两文件移入并改名为实操季编号，如 `op-00-full-journey.md` / `-outline.md`（前缀 `op-` 待路线图定）。
2.3 S2-00 正文/outline 内"第二季从这里开始"等导语**语义正确无需翻转**（它本就是实操季开篇），仅核对季号措辞。
2.4 注意 S2-00 图片引用仍为 `](../pic/`，目录平级，无需改。

### 步骤 3：细节季 12 篇正文的季号字样修正

> 验证：`grep -rn "第二季" 文章系列/第三季-skill深度剖析/` 仅剩语义正确的内容引用，无错误季号。

3.1 批量把 frontmatter description 与导语里的"第二季" → "第三季"（21 文件，s3-01~s3-11）。
3.2 顺手修正 outline description 的"第 N 篇"与文件编号对齐（s3-02/s3-03/s3-05/s3-10/s3-11 等历史错号）。
3.3 各篇结尾"下一篇"预告若引用兄弟篇，核对编号链不断。

### 步骤 4：更新 006 顶层总规划

> 验证：006 全文无"第二季=skill剖析"旧叙事；三季表、对账表、资产节奏、下一步一致。

4.1 改写第一节总叙事弧线：思想 → 实操 → 细节。
4.2 改写第二节三季总览：第二季=项目实操（场景矩阵）、第三季=细节设计（skill 剖析 + 进阶/治理专题）。
4.3 第二节"第一季篇目演化说明"表里的"升级吸收进 S2-xx"指针改为新季号（细节季 s3-xx）。
4.4 第三节对账表：005 编号列说明改为"现细节季 s3-xx"；端到端真实跑一圈（原 S2-00）状态改为"上移实操季总览"。
4.5 第四节"第三季目录"：改为指向新的实操季路线图 + 细节季说明；007 的进阶/治理内容标注为"下沉细节季专题"。
4.6 第五节资产节奏、第六节下一步：按新三季更新。
4.7 关联规划补 brainstorm 与本 plan 链接。

### 步骤 5：处理 007 + 新建实操季路线图

> 验证：007 不再自称"第三季独立路线图"；实操季路线图含场景矩阵 5 篇。

5.1 新建 `运营规划/008-第二季实操路线图.md`（编号续 007）：含场景矩阵（实操-00 总览 / 01 日常增量 / 02 从0到1 / 03 存量改造 / 04 多仓多端），对齐 brainstorm 的矩阵表与两张地图。
5.2 007 重定位：标题/定位改为「第三季细节季的进阶与治理专题清单」，或并入 006 后废止——计划倾向**保留 007 但改定位为"细节季专题选题池"**，避免再造悬空。
5.3 005（现第二季路线图）：加演化说明，指出其内容已平移为第三季细节季，季号以 006 为准。

### 步骤 6：交叉引用与导航同步

> 验证：`grep -rn "第二季-skill深度剖析" --include="*.md" 文章系列/` 为 0；运营规划 README、台账、顶层 README 季号一致。

6.1 `文章系列/README.md`：目录结构树 + 各季小节，季名/路径更新（第二季→实操、第三季→细节）。
6.2 `文章系列/已发布台账.md`：第二季/第三季表头与路径指针更新。
6.3 `运营规划/README.md`：编号清单加 008、季对应树更新。
6.4 `运营规划/003-系统总框架.md`：引用 skill 剖析目录处更新。
6.5 `skills/spec-wechat-publish/SKILL.md`：argument-hint 示例、路线图指针核对（扫描已递归，无需改逻辑）。

### 步骤 7：CHANGELOG + 全局核验

7.1 追加一条 `(user-visible)` CHANGELOG 记录，概述三季重构。

## 风险与回滚

| 风险 | 缓解 |
|---|---|
| 跨目录移动 S2-00 时图片/引用断链 | 图片为相对 `../pic/` 平级，移动后仍在 `文章系列/` 下；移动后立即抽查 |
| 文件批量改名遗漏 outline | 步骤 1.4 用 `grep -c '^s2-'` 卡值为 0 |
| 006 大改引入自相矛盾 | 改完用 doc-review 过一遍内部一致性 |
| 中文路径在 zsh 下 git mv/glob 异常 | 用绝对路径单文件操作，避免变量分词（上轮已踩坑） |
| 历史错号修正引入新错 | 改 outline 编号时对照文件名逐一核对 |

回滚：所有改动在工作区未提交，`git checkout -- .` + 删除新建目录即可还原。
