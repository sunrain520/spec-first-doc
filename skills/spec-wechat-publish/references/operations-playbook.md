# 运营质量 playbook（skill 内引用）

> 本文件是 spec-wechat-publish skill 的运营质量引用文档，供 **Step 4.7 标题运营检查** 和 **Step 6.6 运营质量检查** 读取。
> 完整经验来源：`docs/solutions/conventions/wechat-article-operations-playbook-2026-06-15.md`。
> skill 已有的**硬性 gate**（标题前缀、≤120 字摘要、≥1.5 万字、6-10 配图、版式）在 Step 4.7 / 5.5 强制；本文件管的是 gate 之外的**运营软质量**——打开率、完读率、转化、传播。

## 一、标题运营检查（配合 Step 4.7 的硬格式校验）

硬格式（Step 4.7 已校验）：`Spec-First：`（全角冒号）开头、无正文 H1。

运营软质量（本文件补充）：
- **冒号后那段写读者痛点/收益/悬念，不写作者成就。** 前缀不可省（品牌辨识），但冒号后是拉打开率的主力。
- 自检问句：标题在说"你正踩什么坑 / 你会得到什么"，还是"我做了什么"？后者要改。
- **冒号后控制在 13-20 字**（移动端列表只显前 37-48 字符，加 `Spec-First：` 前缀后更易截断；核心词前置、强词放两端）。
- **收益/悬念要具体**：用数字/技术名/方法（"延迟降 40%""从 REST 迁到 gRPC"）替代泛泛"更快/更好"；悬念点出问题但不剧透解法，不做纯疑问句标题党。
- 对技术受众**不用夸大式紧迫**（"最后机会""不看后悔"）——会触发不信任、抬高取关；紧迫感要落在可兑现的真实技术事实上。
- 定稿前**准备 2-3 个不同框架（痛点/收益/悬念）的候选标题**对照择优，不一稿定（读标题的人是读正文的 5-8 倍，损耗主要在"打开前"）。
> 外部依据：Backlinko / Invesp / Buffer(Ogilvy·Copyblogger) / NN/g。

| ❌ 作者成就 | ✅ 读者痛点/收益 |
|---|---|
| `Spec-First：我用 spec-first 跑完了整个流程` | `Spec-First：同样用 AI 写代码，为什么有人越改越顺、有人越改越崩` |
| `Spec-First：如何用 AI 调试 bug` | `Spec-First：AI 改崩了怎么收拾——不是"你再检查一下"能解决的` |

短钩子文章豁免 `Spec-First：` 前缀（用口语化钩子拉泛 AI 读者）。

## 二、开篇钩子四步（首屏完读率）

首屏（前 3 屏）决定去留。新文章开头应完成：

1. **加粗金句**——一句话核心张力。
2. **1-2 个真实痛点故事**——有人、有事、有卡壳现场，让读者看到自己。
3. **点破张力**——"差别不在 X，在 Y"。
4. **正面体感故事**——站对位置后的画面感（可选但推荐）。

反模式：用 `> 导读` + 专有名词（如"本文讲解 X 的三种模式和 Y 拓扑"）起手 → 新读者无代入感直接划走。导读可以有，但放在痛点故事**之后**，且声明"没读过前序也不影响"。

**情感弹药前置**：画面感故事、翻车现场不要囤在结尾 §N，前置到开头；结尾用回收深化而非首次叙述。

**外部数据支撑的硬约束（NN/g 眼动/可用性研究）：**
- **倒金字塔**：首段就给核心结论/价值，不铺垫背景。自检——只读首段是否就知道这篇解决什么、给什么答案。
- **假设读者只读全文约 20%**（实测 18-28%）：核心价值压进最初几百字。自检——读完开头 20% 字数离开，是否已拿到主要收获。
- **钩子进前两屏**：74% 浏览时间在前两屏、42% 在顶部 20%；最有价值的钩子（核心问题/关键数据/效果预览）必须出现在这。
- **段首前两词测试**：79% 用户只扫描；句首/段首/标题头几个词必须直接表意。自检——遮住每段首句前两个词，是否仍能猜到该段讲什么。
- **去营销腔**：删"史上最强/颠覆性"等无法被事实支撑的最高级（仅去营销腔即提升可用性 27%），前几屏放真实数字/来源作"值得继续读"的信号。

## 三、结尾 CTA（转化 + 传播）

主干正文结尾必须包含（缺一即转化漏损）：

1. **关注引导**（不佛系）：「怕错过的话，关注一下不迷路。」
2. **转发钩子**（给读者一个替朋友转的理由 = 社交货币/实用价值）：「如果你身边也有人……，把这个转给他。」
3. **阅读原文话术 + 链接**：「想直接看代码或装来试，文末『阅读原文』直达 GitHub。」
4. **下一篇预告**（降门槛引导先读）：blockquote 给下篇标题 + 一句话钩子。

**主次分明（外部：单一主 CTA 比多 CTA 点击高 371%，3 个以上下降）：** 公众号语境下"关注/转发/阅读原文"是平台固定动作，但**四件套不要平铺等权**——按本篇目标定一个**主 CTA**（拉新期→关注追更；导流期→阅读原文去 GitHub），其余作次级，主 CTA 用更强的动词+收益措辞（"动词+收益"比笼统措辞转化高一倍以上），并紧跟一句降风险/建信任的话（开源/免费/已能用）。

反模式：佛系「欢迎试用、提 issue」收尾；CTA 堆一起无引导语、无主次；缺转发钩子（公众号传播主靠转发，不给理由就不转）。

## 三·五、系列连载承接与追更

连载靠"未闭合的认知张力"驱动追更（蔡格尼克效应）+ 可预期节奏养成习惯：

- **下一篇预告留悬念不剧透**：点名下一篇要回答的**那个具体问题/要揭晓的具体结果**（不是泛泛"敬请期待"），但不给结论或完整路径——说全了张力就消散，追更动机没了。
- **开头一句"上集回顾"**：一两句承接上一篇，同时服务返场老读者和首次进来的新读者；配合系列命名/编号/前后篇互链显性化系列结构。
- **单篇自成价值 + 长文承载纵深**：每篇让新读者独立读懂（不依赖前序也能拿到收获），靠 1.5 万字深度留住老读者，避免中段稀薄掉队。
- **固定可预期的发布节奏**：把追更变成约定式习惯；用每篇留存/承接数据复盘驱动下一篇选题与钩子。
> 外部依据：Psychology Today(Zeigarnik) / Orbit Media 2025。

## 四、运营优化原则（漏斗四层自检）

- **打开率**：标题命中当下痛点/承诺收益、≤20 字、收益具体（见上）。
- **完读率**：倒金字塔首段给价值；钩子进前两屏；情感弹药前置；段落短（移动端单段 ≤ 3 行）、一段一观点、关键词在段首；可枚举内容改要点列表（提升可用性 47%）；每隔几屏有列表/配图/代码块当视觉落点。
- **转化**：CTA 完整且主次分明（一个主 CTA + 动词收益措辞 + 降风险句）。
- **传播**：做一张"全景一图流"可截图总图当传播资产（开头引导「建议先截图存下」）；加粗金句可独立传播；表格/对比图/结论卡适合截图（可备竖版 1080×1350 / 方版 1080×1080 / 预览图 1200×630）。

## 五、读者分层（一篇锁定一类主受众）

- A 型（认同方法论）→ 第一季；B 型（想确认能用）→ 第二季；进阶/团队（用深用稳）→ 第三季。
- **一篇只锁定一类主受众**，开头就对他说话。不要同时服务三类人（焦点模糊、篇幅膨胀）。

## 六、字数约束

内容为主，**不为压缩而过度精简**——1.5 万字是深度文的留存利器，不是负担。顶部拉新交给短钩子，不靠砍正文。

## 运营质量检查清单（Step 6.6 用）

写完/润色后，对照逐条核（软性，不阻断发布，但应提示用户）：

- [ ] 标题冒号后是读者痛点/收益，不是作者成就；≤20 字、含一个具体数字/技术名/方法、无夸大式紧迫
- [ ] 首屏是痛点故事开场，不是导读+专名；只读首段就知道这篇解决什么、给什么答案（倒金字塔）
- [ ] 最有价值的钩子在前两屏；读完开头 20% 字数就离开也已拿到主要收获
- [ ] 情感弹药（画面感故事）在开头出现过，没全囤在结尾
- [ ] 一段一观点、段首前两词能猜出段意；可枚举内容已改要点列表；每隔几屏有视觉落点
- [ ] 结尾 CTA 完整（关注+转发+阅读原文+下篇预告）且主次分明：有一个主 CTA、动词+收益措辞、紧跟降风险句
- [ ] 下一篇预告点名了具体问题/结果但不剧透；开头有一句"上集回顾"承接前篇
- [ ] 有可截图传播的资产（全景图/核心对比表/金句/结论卡）
- [ ] 本篇锁定了一类主受众，没同时服务三类
- [ ] 加粗金句可独立传播（脱离上下文仍有力）
