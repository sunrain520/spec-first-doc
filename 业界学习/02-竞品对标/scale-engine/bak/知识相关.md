# SCALE Engine 知识相关说明

更新日期：2026-06-02

本文单独梳理 `scale-engine` 中“知识相关”的能力：知识库层、记忆层、上下文层、代码图谱层、Skill/Agent/组件依赖、安装阶段，以及这些能力在 workflow 各节点中的使用方式。

## 总览

`scale-engine` 的知识体系不是一个单独模块，而是一组分层能力：

```text
Agent Adapter / Session 注入
  -> Context Pack / Context Compiler
  -> Memory Provider Router
  -> Memory Fabric / Memory Brain
  -> KnowledgeBase / GraphifyKnowledgeBase
  -> CodeGraph / Graphify / fallback scan
  -> Skill Radar / Skill Policy
  -> Out-of-scope / Cerebrum / Glossary
```

核心原则：

- `define/explore`、`plan`、`review` 主要读知识。
- `build/execute` 主要消费知识约束并产生 runtime evidence。
- `verify` 主要验证知识、provider 和结论是否可信。
- `ship/consolidate` 才做长期知识沉淀。
- 外部 memory provider 默认只读，长期写入需要显式策略和评审边界。

## Workflow 节点如何使用知识

| Workflow 节点 | 别名 | 如何使用知识相关层 | 主要读取 | 主要写入/沉淀 | 常用命令/证据 |
| --- | --- | --- | --- | --- | --- |
| `define` | `explore` | 用上下文、记忆和代码图谱降低歧义，确认术语、历史经验、已拒绝概念和影响面 | Glossary、Context Pack、Memory Provider recall、Memory Fabric、CodeGraph/Graphify、Out-of-scope、Cerebrum | 通常不写长期知识，只记录探索证据 | `scale context pack`、`scale memory provider recall`、`scale memory pack`、`scale codegraph query`、`scale out-of-scope check`、`scale skill radar` |
| `plan` | `plan` | 把召回结果转成计划约束：历史经验、out-of-scope、影响面、skill/provider 和验证门禁 | Context Compiler、Memory recall、Memory Fabric、CodeGraph impact/context、Skill Policy、Cerebrum | 计划产物；必要时记录新的 out-of-scope rationale | `scale ai-os plan`、`scale codegraph impact`、`scale codegraph context`、`scale skill plan` |
| `build` | `execute` | 消费计划中的知识约束，按图谱推荐文件局部读取，遵守 do-not-repeat/preference，产生 runtime evidence | 计划产物、CodeGraph context files、Cerebrum、Context Pack、docs/ADR/glossary、Skill Plan | Runtime Evidence、Tool/skill evidence；必要时更新长期文档或 ADR | `scale context status`、实际命令证据、`skill-evidence.md` |
| `verify` | `verify` | 验证知识是否可用、结论是否可信，检查 provider/context budget/memory pack/codegraph 状态 | Memory Fabric、Memory Provider status、Code Intelligence status、Context Budget、Runtime Evidence | Verification evidence、失败证据 | `scale context doctor`、`scale memory doctor`、`scale memory provider status`、`scale codegraph status`、`scale tool doctor` |
| `review` | `review/security` | 复查记忆、图谱、证据和历史事故，发现冲突、遗漏影响面、安全/质量风险 | Runtime Evidence、Memory Pack、Memory Brain、Memory contradictions、CodeGraph impact/context、Security/standards docs | Review findings、accepted risk；可把拒绝原因写入 out-of-scope | `scale memory query`、`scale memory contradictions`、`scale memory dream`、`scale codegraph impact`、`scale out-of-scope add/check` |
| `ship` | `consolidate` | 做知识收口：把真实证据压缩成 learning candidate，经人审后进入 Memory Brain、docs、rules 或 ADR | Runtime Evidence、Memory Fabric pack、Review/verification evidence、Cerebrum、Out-of-scope、docs/ADR/glossary | `.scale/memory/learning-candidates/*`、Memory Brain active node、`summary.md`、长期 docs/rules/ADR | `scale memory settle`、`scale memory promote`、`scale memory cerebrum --write`、`scale out-of-scope add` |

使用原则：

| 原则 | 含义 |
| --- | --- |
| 先读后写 | `define/plan/review` 主要读知识；`ship/consolidate` 才沉淀稳定知识 |
| 证据优先 | 推荐闭环是 `runtime evidence -> memory pack -> memory settle -> 人审 -> knowledge/docs/rules` |
| Provider 有 fallback | Memory 默认 `gbrain -> agentmemory -> scale-local`，`requireEvidence=true`、`maxResultsPerProvider=5`；Code Intelligence 默认 `codegraph/graphify -> internal-scan/rg/read` |
| 预算可解释 | Context Compiler 和 Memory Fabric 记录 included/omitted、token 预算和裁剪原因 |
| 外部写入默认关闭 | `.scale/memory-providers.json` 当前 `allowExternalWrite=false` |

## 需求阶段可使用的知识

需求阶段对应 `define/explore`，目标是澄清需求、理解项目约束、降低歧义和识别主要矛盾。这个阶段主要读知识，不直接沉淀长期记忆。

| 可用知识 | 作用 | 常用入口 | 输出/证据 |
| --- | --- | --- | --- |
| 项目基础上下文 | 了解项目目标、运行方式、开发规范、工作流入口 | `README.md`、`AGENTS.md`、`docs/guides/GETTING_STARTED.md`、`docs/workflow/README.md` | 已读文件清单、main contradiction、`explore.md` |
| 术语和项目语言 | 确认项目专有词和概念边界 | `.scale/GLOSSARY.md`、`scale context glossary` | 术语引用、歧义点、待确认问题 |
| Context Pack | 生成任务相关预算化上下文 | `scale context pack --task "..." --files "..." --level M` | included/omitted sections、token budget |
| Memory Provider 召回 | 查询跨会话记忆、历史经验、相似问题 | `scale memory provider recall "..." --json` | provider order、selected provider、fallback reason、recall items |
| Memory Fabric | 聚合 runtime evidence、session events、knowledge recall、project graph 摘要 | `scale memory pack --task "..." --budget 4000` | memory context pack |
| KnowledgeBase 已验证经验 | 召回 lesson、规则、历史教训 | 通过 `MemoryFabric` 或 `KnowledgeBase.recall/recallByVector` | Top K、relevance、verified 标记 |
| CodeGraph / Graphify | 理解代码结构、模块关系、符号、路由、影响面 | `scale codegraph query "..."`、`scale codegraph context --symbol ...` | 命中文件、符号、confidence、fallbackUsed |
| Out-of-scope 制度记忆 | 查询需求是否以前已被拒绝 | `scale out-of-scope check "..."` | matched entry、拒绝原因 |
| Cerebrum 偏好/禁忌 | 读取 preference 和 do-not-repeat 规则 | `scale memory cerebrum`、`.scale/cerebrum.md` | 偏好/禁忌摘要 |
| Skill Radar / Skill Policy | 判断需求涉及哪些能力、skill、MCP、CLI、provider | `scale skill radar --task "..." --phase define`、`scale skill plan ...` | `skill-plan.md`、required/recommended skills |
| 外部事实和网页信息 | 需求依赖最新 API、法规、产品、竞品或网页动态内容时使用 | `web-access`、`agent-browser`、官方文档/来源引用 | source citation、browser evidence、未确认项 |

推荐顺序：

1. 读 `README.md`、`AGENTS.md`、`.scale/GLOSSARY.md` 和 workflow 文档。
2. 生成任务 context：`scale context pack --task "..."`。
3. 查历史经验：`scale memory provider recall "..."` 或 `scale memory pack --task "..."`。
4. 查代码影响面：`scale codegraph query "..."` 或 `scale codegraph context --symbol ...`。
5. 查 out-of-scope 和 Cerebrum。
6. 用 Skill Radar/Skill Plan 判断后续 `plan/build/verify` 需要哪些 skill 和证据。

阶段边界：

- 可以输出 `explore.md`、`mini-prd.md`、`skill-plan.md`、已读文件清单、问题清单和未确认项。
- 不应直接 promote Memory Brain，也不应把一次需求讨论中的猜测写入长期规则。
- 稳定可复用的新事实先作为候选记录，等 `ship/consolidate` 阶段通过 `memory settle -> 人审 -> promote/docs/rules` 再沉淀。

## 安装和初始化阶段

这里的“安装阶段”不是 workflow 的 `build/execute` 节点，而是项目接入和第三方依赖准备阶段。业务 workflow 节点默认只消费能力；真正安装或初始化通常发生在 `scale init`、`scale setup`、`scale bootstrap deps`、`scale codegraph init` 或 provider 自身初始化命令里。

`setup`/`bootstrap deps` 默认先输出计划，只有显式 `--apply` 或 `--yes` 才执行安装。

| 对象 | 类型 | 安装/初始化阶段 | 触发命令 | 安装/构建内容 | 后续验证 |
| --- | --- | --- | --- | --- | --- |
| Agent adapter 知识文档 | Agent 宿主接入 | 项目初始化阶段 | `scale init --agent <agent>` | 生成 settings、knowledge doc、skills dir、hooks、规则文件 | 查看 init 输出的 `Knowledge:` 路径；`scale context inject/status` |
| `.scale/config.yaml` | 项目配置 | 项目初始化阶段 | `scale init ...` | 写入 profile、storage、knowledge backend、memory provider | `scale doctor`、`scale setup --verify` |
| GBrain | memory provider / CLI | 第三方依赖安装阶段，`memory` pack | `scale setup --pack memory --memory-provider gbrain --apply --yes` | 安装 `gbrain` CLI；执行 bootstrap 初始化 `gbrain init --pglite --no-embedding`；写入 `.scale/memory-providers.json` 顺序 | `scale memory provider status --json`、`scale tool doctor --tools gbrain --json`、`npm run smoke:gbrain` |
| agentmemory | optional memory provider / MCP | 手动外部服务接入阶段 | 配置 provider 或按上游命令启动 agentmemory/MCP | 可选外部 provider；当前默认禁用 | `scale memory provider status --json`；隐私/保留/删除边界评审 |
| scale-local / Memory Brain | 本地长期记忆 | 运行时懒创建 | `scale memory ingest/query/promote/...` | 创建 `.scale/memory/brain.sqlite` schema | `scale memory query`、`scale memory dream`、`scale memory contradictions` |
| Memory Fabric | 本地 context pack 构建 | workflow 使用阶段，无第三方安装 | `scale memory pack --task ...` | 聚合 evidence/events/knowledge/graph 摘要 | `scale memory doctor` |
| Graphify CLI | knowledge graph provider / CLI / skill | 第三方依赖安装阶段，`knowledge` pack | `scale setup --pack knowledge --apply --yes` | 安装 Graphify；执行 `graphify install --platform codex`、`graphify hook install` | `scale tool doctor --tools graphify --json`、`graphify hook status` |
| Graphify 图谱产物 | 知识图谱 artifact | provider 初始化/图谱构建阶段 | `graphify update <project> --no-cluster` | 生成 `graphify-out/graph.json` 和报告 | `scale codegraph status --json`、`npm run smoke:graphify -- --large-project <path>` |

| `.scale/code-intelligence.json` | 代码智能 provider 配置 | knowledge pack post-action 或手动 init | `scale codegraph init`；`scale bootstrap deps --pack knowledge --apply` | 写入 `codegraph`、`graphify` provider 和 fallback | `scale codegraph status --json` |
| `.scale/memory-providers.json` | memory provider 路由配置 | memory pack post-action 或 provider use | `scale memory provider init`；`scale memory provider use gbrain --json` | 写入 provider 顺序、写入策略、安全边界；默认 `allowExternalWrite=false`、`requireEvidence=true`、`maxResultsPerProvider=5` | `scale memory provider status --json` |
| UI/文档知识 skills | skill-file / vendor reference | UI pack 安装阶段 | `scale setup --pack ui --include awesome-design-md --apply`、`scale setup --pack ui --include ui-ux-pro-max --apply` | 同步到 `~/.scale/vendor/*`，生成 `~/.agents/skills/*/SKILL.md` | `scale tool doctor --tools awesome-design-md,ui-ux-pro-max --json`、`scale skill doctor --json` |
| Skill Radar / Skill Policy | 内置路由能力 | 项目配置阶段，无外部安装 | `.scale/skills.json` | 推荐 required/recommended skills 和证据 | `scale skill radar --task ...`、`scale skill plan ...` |
| Out-of-scope | 本地制度记忆 | 使用时创建 | `scale out-of-scope add ...` | 生成 `.scale/out-of-scope/` 条目 | `scale out-of-scope list/check ...` |
| Cerebrum | 本地偏好/禁忌知识 | 使用时写入 | `scale memory cerebrum --type preference ...` 或 `--type do-not-repeat ...` | 写入 KnowledgeBase 条目，可生成 `.scale/cerebrum.md` | `scale memory cerebrum --json` |

安装顺序建议：

1. 项目初始化：`scale init --agent <agent>`。
2. 依赖计划：`scale setup --pack memory,knowledge --json` 或 `scale bootstrap deps --pack memory,knowledge`。
3. 显式安装：`scale setup --pack memory,knowledge --apply --yes`。
4. Provider 初始化：确认 bootstrap 阶段执行 `gbrain init --pglite --no-embedding`、`graphify install --platform codex`、`graphify hook install`、`graphify update <project> --no-cluster`、`codegraph init -i <project>`。
5. 状态验证：`scale setup --verify`、`scale memory provider status --json`、`scale codegraph status --json`、`scale tool doctor --tools gbrain,codegraph,graphify --json`、`scale doctor`。
6. Workflow 使用：`scale context pack`、`scale memory provider recall`、`scale memory pack`、`scale codegraph query/context/impact`。
7. 交付沉淀：`scale memory settle`、`scale memory promote`、docs/rules/ADR/out-of-scope。
