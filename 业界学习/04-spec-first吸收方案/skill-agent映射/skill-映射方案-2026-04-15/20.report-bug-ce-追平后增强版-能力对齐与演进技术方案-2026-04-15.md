## Part A：上游基线与代码事实

### A.1 核对范围

- source:
  - `plugins/compound-engineering/skills/report-bug-ce/SKILL.md`
- target:
  - `skills/report-bug/SKILL.md`

### A.2 代码事实

该映射没有 supporting files，也没有额外脚本。差异全部集中在一个 `SKILL.md`。

### A.3 关键差异

#### A.3.1 品牌与仓库目标迁移

从代码可见，主要差异都是品牌/目标仓库迁移：

- `report-bug-ce` → `report-bug`
- `compound-engineering plugin` → `spec-first`
- `EveryInc/compound-engineering-plugin` → `sunrain520/spec-first`
- issue title / label / success message 同步迁移

#### A.3.2 版本来源调整

上游让用户从 plugin registry/installed_plugins 读 plugin version。

当前版本收敛为从以下来源读取：

- 当前项目 `package.json`
- bundled `.claude-plugin/plugin.json`
- installed package metadata

这更符合 `spec-first` 的 npm CLI + packaged assets 形态。

### A.4 现状判断

#### 结论

这是一个**能力同构、实现轻差异**的映射。

差异本质是：

1. 品牌替换
2. 目标 GitHub 仓库替换
3. 版本来源从 plugin registry 转为 CLI/package metadata

没有发现结构性能力缺口。
