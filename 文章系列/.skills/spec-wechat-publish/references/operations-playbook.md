# 运营质量 playbook（skill 内引用）

> 本文件是 spec-wechat-publish skill 的运营质量引用文档，供 **Step 4.7 标题运营检查** 和 **Step 6.6 运营质量检查** 读取。
> 完整经验来源：`docs/solutions/conventions/wechat-article-operations-playbook-2026-06-15.md`。
> skill 已有的**硬性 gate**（标题前缀、≤120 字摘要、≥1.5 万字、6-10 配图、版式）在 Step 4.7 / 5.5 强制；本文件管的是 gate 之外的**运营软质量**——打开率、完读率、转化、传播。

## 一、标题运营检查（配合 Step 4.7 的硬格式校验）

硬格式（Step 4.7 已校验）：`Spec-First：`（全角冒号）开头、无正文 H1。

运营软质量（本文件补充）：
- **冒号后那段写读者痛点/收益/悬念，不写作者成就。** 前缀不可省（品牌辨识），但冒号后是拉打开率的主力。
- 自检问句：标题在说"你正踩什么坑 / 你会得到什么"，还是"我做了什么"？后者要改。

| ❌ 作者成就 | ✅ 读者痛点/收益 |
|---|---|
| `Spec-First：我用 spec-first 跑完了整个流程` | `Spec-First：同样用 AI 写代码，为什么有人越改越顺、有人越改越崩` |
| `Spec-First：如何用 AI 调试 bug` | `Spec-First：AI 改崩了怎么收拾——不是"你再检查一下"能解决的` |

短钩子文章豁免 `Spec-First：` 前缀（用口语化钩子拉泛 AI 读者）。

## 二、开篇钩子四步（首屏完读率）

首屏（前 3 屏）决定去留。新文章开头应完成：

1. **加粗金句**——一句话核心张力。
2. **1-2 个真实痛点故事**——有人、有事、有卡壳现场，让读者看到自己。
3. **点破张力**——"差别不在 X，在 Y"。
4. **正面体感故事**——站对位置后的画面感（可选但推荐）。

反模式：用 `> 导读` + 专有名词（如"本文讲解 X 的三种模式和 Y 拓扑"）起手 → 新读者无代入感直接划走。导读可以有，但放在痛点故事**之后**，且声明"没读过前序也不影响"。

**情感弹药前置**：画面感故事、翻车现场不要囤在结尾 §N，前置到开头；结尾用回收深化而非首次叙述。

## 三、结尾 CTA 三件套（转化 + 传播）

主干正文结尾必须包含（缺一即转化漏损）：

1. **关注引导**（不佛系）：「怕错过的话，关注一下不迷路。」
2. **转发钩子**（给读者一个替朋友转的理由）：「如果你身边也有人……，把这个转给他。」
3. **阅读原文话术 + 链接**：「想直接看代码或装来试，文末『阅读原文』直达 GitHub。」
4. **下一篇预告**（降门槛引导先读）：blockquote 给下篇标题 + 一句话钩子。

反模式：佛系「欢迎试用、提 issue」收尾；CTA 堆一起无引导语；缺转发钩子（公众号传播主靠转发，不给理由就不转）。

## 四、运营优化原则（漏斗四层自检）

- **打开率**：标题命中当下痛点/承诺收益（见上）。
- **完读率**：痛点前置首屏；情感弹药前置；段落短（移动端单段 ≤ 3 行）；每 H2 考虑一张配图打断疲劳。
- **转化**：CTA 三件套完整，给具体行动指令。
- **传播**：做一张"全景一图流"可截图总图当传播资产（开头引导「建议先截图存下」）；加粗金句可独立传播；表格/对比图适合截图。

## 五、读者分层（一篇锁定一类主受众）

- A 型（认同方法论）→ 第一季；B 型（想确认能用）→ 第二季；进阶/团队（用深用稳）→ 第三季。
- **一篇只锁定一类主受众**，开头就对他说话。不要同时服务三类人（焦点模糊、篇幅膨胀）。

## 六、字数约束

内容为主，**不为压缩而过度精简**——1.5 万字是深度文的留存利器，不是负担。顶部拉新交给短钩子，不靠砍正文。

## 运营质量检查清单（Step 6.6 用）

写完/润色后，对照逐条核（软性，不阻断发布，但应提示用户）：

- [ ] 标题冒号后是读者痛点/收益，不是作者成就
- [ ] 首屏是痛点故事开场，不是导读+专名
- [ ] 情感弹药（画面感故事）在开头出现过，没全囤在结尾
- [ ] 结尾有关注引导 + 转发钩子 + 阅读原文话术 + 下篇预告
- [ ] 有可截图传播的资产（全景图/核心对比表/金句）
- [ ] 本篇锁定了一类主受众，没同时服务三类
- [ ] 加粗金句可独立传播（脱离上下文仍有力）
